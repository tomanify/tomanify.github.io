نصب و راه‌اندازی Tomanify

این صفحه برای راه‌اندازی استاندارد است: مناسب کسی که می‌خواهد افزونه را نصب کند، منبع نرخ را وارد کند، اولین محصول خارجی را بسازد و بعد بروزرسانی‌ها را خودکار کند.

قبل از شروع

  • ووکامرس باید فعال باشد.
  • بهتر است تصمیم بگیرید واحد نهایی سایت تومان باشد یا ریال.
  • یک منبع نرخ HTTPS آماده داشته باشید؛ ساده‌ترین گزینه، JSON بازار آزاد است.
  • اگر فروشگاه حساسی دارید، از همین ابتدا دربارهٔ Margin و منابع پشتیبان فکر کنید.

لینک پیشنهادی شروع سریع

برای اولین تست، این لینک را در Sources وارد کنید:

https://raw.githubusercontent.com/rate-json/default/main/data.json

بعد از آن، داشبورد افزونه را باز کنید و بروزرسانی نرخ‌ها را اجرا کنید.

مسیر راه‌اندازی در ۶ قدم

  1. نصب و فعال‌سازی: افزونه را نصب و فعال کنید.
  2. General: واحد، زبان و بازهٔ Cron را بررسی کنید.
  3. Sources: JSON یا RSS/XML را وارد کنید.
  4. Dashboard: روی Update Rates بزنید و آخرین منبع/زمان را ببینید.
  5. Product: اولین محصول را foreign کنید و قیمت خرید/ارز/Margin را وارد کنید.
  6. Validation: قیمت نهایی، مرتب‌سازی و در صورت نیاز شورتکد را تست کنید.

سوالات متداول این صفحه

جواب کوتاه به سوال‌هایی که معمولاً قبل از نصب، هنگام تنظیمات، یا زمان عیب‌یابی پیش می‌آید.

بعد از نصب، اولین کار چیست؟

منبع نرخ را در تب Sources تنظیم کنید، بعد از داشبورد Update Rates بزنید و سپس اولین محصول foreign را تست کنید.

اگر هنوز URL منبع را وارد نکرده باشم چه می‌شود؟

افزونه به‌صورت پیش‌فرض هیچ URL آماده‌ای را داخل تنظیمات قرار نمی‌دهد؛ بنابراین تا وقتی شما آدرس را وارد نکنید، درخواست خارجی هم انجام نمی‌شود.

آیا WP-Cron همیشه دقیق اجرا می‌شود؟

در هاست اشتراکی ممکن است به بازدید سایت وابسته باشد. اگر زمان‌بندی دقیق‌تری می‌خواهید، از Cron واقعی هاست یا سرویس cron استفاده کنید.

تغییر واحد از تومان به ریال دوباره نرخ را می‌گیرد؟

خیر. فقط قیمت ووکامرس از روی پایه تومان بازنویسی می‌شود و تبدیل دوباره انجام نمی‌شود.

برای شروع Margin را روی چه عددی بگذارم؟

برای اکثر فروشگاه‌هایی که با JSON رایگان بازار آزاد شروع می‌کنند، Margin محافظه‌کارانه مثل ۳٪ تا ۵٪ می‌تواند ریسک عقب‌ماندن نرخ را پوشش دهد.
